# 📅 个人计划（月视图）

> 一个简洁、本地优先的月度计划与待办管理工具，无需注册，打开即用。支持任务管理、周目标设定、每日感想记录、多格式导出和暗色模式。

---

## ✨ 功能特性

- 📆 **月视图管理**：以月为单位查看和管理任务，支持拖拽移动任务（桌面端）
- 🎯 **周目标设定**：每周设定核心目标，追踪完成进度
- 💭 **每日感想**：记录每日反思与收获，支持编辑删除
- ✅ **任务完成追踪**：实时显示每日完成度，100% 触发礼花效果
- 🔄 **任务拖拽排序**：支持鼠标拖拽和移动端触摸拖拽
- 🖱️ **右键菜单**：任务支持右键快速编辑、删除、移动
- 🌓 **暗色模式**：护眼模式，手动切换或跟随系统
- 📤 **多格式导出**：支持 JSON 备份、Markdown 报告、Word 文档、PDF 打印
- 📥 **数据导入**：支持导入 JSON 备份，快速恢复数据
- 🖨️ **打印优化**：支持打印月度报告
- 📱 **响应式设计**：完美适配桌面端、平板、手机

## 📖 使用指南

### 每日使用流程

**🌅 早晨（5分钟）**
- 查看今日任务列表
- 根据周目标安排今日3件要事
- 点击日期格子快速添加新任务

**☀️ 日间（随时）**
- 完成任务时勾选 ✓，自动保存进度
- 临时添加新任务
- 拖拽调整任务优先级

**🌙 晚上（5分钟）**
- 记录今日感想（学到了什么/改进点）
- 回顾未完成任务，调整到明日
- 查看今日完成率，给自己鼓励

### 管理任务

**添加任务**
- 点击日期格子中的 `＋` 按钮
- 点击顶部工具栏的「添加任务」按钮
- 在对话框中填写任务内容

**完成任务**
- 点击任务前的复选框，自动保存并更新进度
- 当日完成率达到100%时会触发礼花效果

**移动任务（桌面端）**
- 直接拖拽任务到其他日期格子
- 使用右键菜单中的「上移/下移」调整顺序

**编辑/删除任务**
- 右键点击任务，选择编辑或删除
- 或使用任务右侧的图标按钮

### 设定周目标
- 在左侧「本月目标」区域输入目标内容
- 点击「添加」或按回车键保存
- 勾选复选框标记目标完成

### 记录每日感想
- 点击月视图中任意日期，选择查看当天感想
- 在文本框中输入感想内容
- 点击「保存感想」或按 `Ctrl+Enter` 保存

### 数据备份与恢复

**导出数据**
1. 点击顶部「导出」按钮
2. 选择导出格式：
   - JSON：完整备份（任务+感想）
   - Markdown：生成月度报告
   - Word：生成 Word 文档
   - PDF：打印为 PDF

**导入数据**
1. 点击「导入」按钮
2. 选择之前导出的 JSON 文件
3. 数据自动恢复

## 💾 数据存储说明

### 存储结构

**任务数据**（键名：`weekly-planner-v1`）
```javascript
{
  "wk:2026-03-16": {
    "goals": [{ id, title, done }],     // 周目标
    "notes": "",                         // 预留字段
    "days": {                            // 每日任务
      "2026-03-16": [{ id, title, done }],
      "2026-03-17": [...]
    }
  }
}
```

**感想数据**（键名：`daily-thoughts-v1`）
```javascript
{
  "2026-03-16": [
    { id, createdAt: 时间戳, content: "感想内容" }
  ]
}
```

### 数据安全
- 所有数据存储在浏览器本地，不上传任何服务器
- 用户完全掌控自己的数据
- 建议定期导出 JSON 文件备份

## 🎨 自定义与扩展

### 修改配色方案
编辑 `style.css` 中的 CSS 变量：

```css
:root {
  --primary: #2563eb;      /* 主色调 */
  --bg: #ffffff;           /* 背景色 */
  --surface: #f8fafc;      /* 面板背景 */
  /* 更多颜色变量... */
}
```

### 添加新功能
- 核心逻辑在 `app.js` 中，按功能模块组织
- 月视图渲染在 `renderMonth()` 函数
- 任务管理相关函数在 `renderTasks()` 等

### 部署为 PWA（可选）
1. 添加 `manifest.json` 文件
2. 编写 Service Worker
3. 通过 HTTPS 部署

## ⚠️ 已知问题与限制

- **数据同步**：数据仅存储在本地，更换设备需手动导入导出
- **历史月份**：月视图支持浏览任意月份，但需手动点击上月/下月按钮
- **大量任务**：当月任务超过 100 条时，月视图滚动可能稍慢
- **跨月周**：周目标以周一为起点，跨月周可能显示不完整
